OERAdap API

El API REST permite adaptar objetos de aprendizaje de forma automática, tan solo necesitas la API key que se te envía al correo de forma gratuita.

Condiciones de uso

La Universidad Politécnica Salesiana conjunto con el grupo de investigación EduTech pone a disposición de esta plataforma de forma gratuita y sin ánimos de lucro, para el desarrollo de la educación equitativa.

Al usar el API el usuario acepta de forma voluntaria los siguientes términos:

Los datos que se guardan son solo información de navegación como información del navegador y datos estadísticos sobre el resumen de lo realizado en esta aplicación con la finalidad re recopilar datos estadísticos para mejorar la aplicación.
Los archivos subidos no se compartirán con ninguna persona o institución pública o privada de igual manera los archivos generados pertenecen solo al dueño de la llave del API por lo que esta no debe ser compartida bajo ningún criterio.
El usuario está en su responsabilidad de subir archivos únicamente recursos educativos (Objetos de Aprendizaje), se admiten solo archivos comprimidos con formato zip.

Peticiones

El API de EduTech OERAdap se basa en principios REST, para acceder a los recursos de los datos o crear los datos se realiza mediante las solicitudes HTTP en formato UTF-8.

Petición Descripción
GET Solicitud de información
POST Solicitud de creación

Parámetros

En la solicitud los parámetros se envían como Query string (cadenas de consultas) es decir adjunto a la URL.

Direccion base de la API: https://oeradap.edutech-project.org/api/v1/oeradapter

Parámetro Descripción Ejemplo
api_key Llave de acceso a la API, puedes obtener llenando el formulario. ?api_key=tu_api_key
adaptation Áreas las que se quiere adaptar el objeto de aprendizaje. ?adaptation=image,audio,video,paragraph,button
file Nombre del parámetro de tipo form-data de tipo file en el que se adjunta el archivo zip form-data

Códigos de estado de respuesta

La API usa los siguientes codigos de respuesta como se define en RFC 2616 y RFC 6585:

Código Descripción
200 Solicitud procesada correctamente.
201 Recurso creado correctamente.
400 Solicitud inválida.
401 Llave de acceso incorrecta.
404 Recurso no encontrado.

Códigos de error de respuesta

Dentro de nuestras respuestas de error establecemos los siguientes códigos para determinar el error.

Código Descripción
invalid_api_key Llave de acceso incorrecta.
file_not_found El recurso solicitado caduco o no existe.
required_file El archivo no se encuentra presente en la petición.
incorrect_format El archivo no es de formato zip, la API solo admite archivos con extensión .zip
learning_Object_Adapted El Objeto de Aprendizaje ya fue adaptado por la herramienta.

Direcciones de la API

La API usa las siguientes direcciones para acceder a los recursos:

Tomando como base la direccions de la API: https://oeradap.edutech-project.org/api/v1/oeradapter

Dirección Método Descripción
/upload POST Path para subir un objeto de aprendizaje se admite solo archivos con formato .zip
/files GET Path para obtener todos los objetos de aprendizajes subidos.
/files/{{"{ID}"}} GET Path para obtener un objeto de aprendizaje subido, mediante su ID.

Direcciones de los datos de respuesta de la API

La API usa las siguientes datos de respuesta

Atributo Tipo Descripción
Id Number Id unico del objeto de aprendizaje.
oa_detail Object Objeto que contiene la informacion basica del objeto de aprendizaje.
created_at Timestamp Fecha de creación de objeto de aprendizaje.
expires_at Timestamp Fecha en la que se eliminara el objeto de aprendizaje del servidor.
preview_origin String Url para visualizar el objeto de aprendizaje original.
preview_adapted String Url para visualizar el objeto de aprendizaje adaptado.
file_detail Object Objeto de información del contenido del objeto de aprendizaje.
adapted_detail Object Objeto de información del contenido adaptado en el objeto de aprendizaje.
config_adaptability Object Objeto de información de la configuracion seleccionada para la optimización.
complete_adaptation Boolean Variable booleana para determinar el estado de la adaptabilidad.
button_adaptation Boolean Variable booleana para determinar el estado de la adaptabilidad del boton o la barra de adaptabilidad.
audio_adaptation Boolean Variable booleana para determinar el estado de la adaptabilidad de los audios.
image_adaptation Boolean Variable booleana para determinar el estado de la adaptabilidad de las images.
paragraph_adaptation Boolean Variable booleana para determinar el estado de la adaptabilidad de los parrafos de texto.
video_adaptation Boolean Variable booleana para determinar el estado de la adaptabilidad de los videos.
file_download String Url de descarga del objeto de aprendizaje adaptado.

Ejemplos subir un objeto de aprendizaje

Para subir un objeto de aprendizaje:

Path: https://oeradap.edutech-project.org/api/v1/oeradapter/upload?api_key=TU_API_KEY&adaptation=image,audio,video,paragraph,button

Método: POST

Parametro en Form-Data: file

Retorno: Retorna un objeto Json al completar la carga, esto puede tardar varios minutos

Ejemplo en Postman
Resultado

El resultado puede tardar dependiendo de la cantidad de datos a adaptar del objeto de aprendizaje en promedio de 5 minutos.

        {{data | json}}

Ejemplos obtener los archivos

Para obtener una lista de los objetos de aprendizaje subidos:

Path: https://oeradap.edutech-project.org/api/v1/oeradapter/files?api_key=TU_API_KEY

Método: GET

Retorno: Retorna un lista de objetos Json

Ejemplo en Postman
Resultado
        {{arrayData | json}}

Ejemplos obtener un archivo por el id

Para obtener un objeto de aprendizaje subido:

Path: https://oeradap.edutech-project.org/api/v1/oeradapter/files/{{"{ID}"}}?api_key=TU_API_KEY

Método: GET

Retorno: Retorna un objeto Json

Ejemplo en Postman
Resultado
        {{data | json}}
